# Oracle-Specific Features in the Codebase

This document lists all Oracle-specific elements currently implemented in the Spring Boot application. These will need to be adapted or replaced during migration to PostgreSQL.

## Current Oracle-Specific Features

### 1. Database Dialect
- **File**: `src/main/resources/application.yml`
- **Feature**: `database-platform: org.hibernate.dialect.OracleDialect`
- **Migration Note**: Change to `org.hibernate.dialect.PostgreSQLDialect` for PostgreSQL.

### 2. Sequences
- **Files**: `src/main/java/com/example/inventory/Product.java`, `src/main/java/com/example/inventory/Inventory.java`
- **Feature**: `@SequenceGenerator` annotations for ID generation.
- **Example**:
  ```java
  @GeneratedValue(strategy = GenerationType.SEQUENCE, generator = "product_seq")
  @SequenceGenerator(name = "product_seq", sequenceName = "product_seq", allocationSize = 1)
  ```
- **Migration Note**: PostgreSQL supports sequences, but syntax differs. Use `SERIAL` or keep sequences.

### 3. Native Queries with Oracle Syntax
- **Files**: `src/main/java/com/example/inventory/ProductRepository.java`, `src/main/java/com/example/inventory/InventoryRepository.java`
- **Features**:
  - `ROWNUM`: Used for pagination (e.g., `WHERE ROWNUM <= ?1`).
  - `NVL`: Null handling (e.g., `NVL(is_active, 0)`).
  - `SYSDATE`: Current timestamp (e.g., `SYSDATE` in queries).
- **Migration Note**: Replace with PostgreSQL equivalents:
  - `ROWNUM` → `LIMIT`/`OFFSET`.
  - `NVL` → `COALESCE`.
  - `SYSDATE` → `CURRENT_TIMESTAMP` or `NOW()`.

### 4. Data Types
- **Files**: `src/main/java/com/example/inventory/Product.java`
- **Feature**: `Boolean` mapped to `NUMBER(1)` in Oracle.
- **Migration Note**: PostgreSQL uses `BOOLEAN` natively.

### 5. Unique Constraints
- **Files**: `src/main/java/com/example/inventory/Inventory.java`
- **Feature**: `@UniqueConstraint` on `product_id` for one-to-one relationship.
- **Migration Note**: PostgreSQL supports unique constraints similarly.

### 6. DDL Auto
- **File**: `src/main/resources/application.yml`
- **Feature**: `ddl-auto: update` (creates/updates tables).
- **Migration Note**: Same in PostgreSQL.

## Proposed Additions for Increased Complexity

To make the migration more realistic, the following Oracle-specific features will be added:

### 1. Stored Procedure
- **Description**: A procedure to calculate total inventory value (sum of price * quantity for active products).
- **Oracle Code**:
  ```sql
  CREATE OR REPLACE PROCEDURE calculate_total_value (total OUT NUMBER) AS
  BEGIN
    SELECT NVL(SUM(p.price * i.quantity), 0) INTO total
    FROM product p
    JOIN inventory i ON p.id = i.product_id
    WHERE p.is_active = 1;
  END;
  ```
- **Migration Note**: Convert to PostgreSQL function/procedure using PL/pgSQL.

### 2. Trigger
- **Description**: An AFTER INSERT/UPDATE trigger on Inventory to log changes to a new AUDIT_LOG table.
- **Oracle Code**:
  ```sql
  CREATE OR REPLACE TRIGGER inventory_audit_trigger
  AFTER INSERT OR UPDATE ON inventory
  FOR EACH ROW
  BEGIN
    INSERT INTO audit_log (table_name, operation, product_id, old_quantity, new_quantity, timestamp)
    VALUES ('inventory', CASE WHEN INSERTING THEN 'INSERT' WHEN UPDATING THEN 'UPDATE' END, :NEW.product_id, :OLD.quantity, :NEW.quantity, SYSDATE);
  END;
  ```
- **Migration Note**: PostgreSQL triggers use different syntax.

### 3. View
- **Description**: A view combining Product and Inventory for active products with stock.
- **Oracle Code**:
  ```sql
  CREATE OR REPLACE VIEW active_products_with_stock AS
  SELECT p.id, p.name, p.price, i.quantity, i.last_updated
  FROM product p
  JOIN inventory i ON p.id = i.product_id
  WHERE p.is_active = 1;
  ```
- **Migration Note**: PostgreSQL views are similar.

### 4. Audit Log Table
- **Description**: New table for logging changes.
- **Oracle Code**:
  ```sql
  CREATE TABLE audit_log (
    id NUMBER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    table_name VARCHAR2(50),
    operation VARCHAR2(10),
    product_id NUMBER,
    old_quantity NUMBER,
    new_quantity NUMBER,
    timestamp TIMESTAMP DEFAULT SYSDATE
  );
  ```
- **Migration Note**: Use PostgreSQL `SERIAL` for ID, `VARCHAR`, `TIMESTAMP`.

### 5. Updated Repositories and Services
- **Description**: Add native queries and stored procedure calls.
- **Migration Note**: Adapt to PostgreSQL syntax.

### 6. Frontend Updates
- **Description**: Add UI for total value and active stock view.
- **Migration Note**: No changes needed, as it's API-driven.
